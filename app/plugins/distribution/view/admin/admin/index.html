依赖插件[ 钱包、分享 ]

1. 分销依赖插件有[ 钱包 、分享 ]，插件可在应用商店下载安装，安装不分先后顺序。

2. 佣金实际返佣至用户钱包余额需要脚本定时执行，访问分销后台也会自动执行一次。

3. 订单提交的同时自动创建分销返佣订单，届时被返佣用户将可在收益明细查看、同时发送站内信。

4. 当订单发生售后金额改变，分佣订单也会重新计算、所以不用担心佣金被多计算问题（注意：仅处理待结算的返佣订单）。

5. 佣金结算请配置[基础数据->佣金结算周期]的配置，默认订单收货时间开始计算30天后结算佣金。

6. 建议在佣金结算周期之后不要再发生订单售后，以免造成资金损失。

7. 支持用户消费向下返佣、仅用户邀请的下级用户获得返佣。
